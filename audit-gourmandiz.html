Audit SEO

Problèmes identifiés

❌
Meta description absente
Critique
Aucune balise meta description n'est définie sur la page d'accueil. C'est le texte qui apparaît sous le titre dans les résultats Google. Sans elle, Google génère un extrait aléatoire, réduisant le taux de clic (CTR).
📸
Images sans attribut alt
Critique
168 images sur 194 (87%) n'ont pas d'attribut alt. Cela impacte le SEO images, l'accessibilité et la compréhension du contenu par Google.
📷 168/194 sans alt
⚠ 87% des images
🔗
Open Graph incomplet
Important
Pas d'og:image définie et l'og:description est juste "Gourmandiz" au lieu d'une vraie description. Les partages sur les réseaux sociaux n'afficheront pas de visuel attractif.
📋
Ratio liens internes/externes
Important
Le site a 133 liens externes contre seulement 53 liens internes. Ce ratio déséquilibré dilue le "link juice" et le maillage interne du site, affaiblissant le SEO on-page.
🔗 53 internes
🌐 133 externes
📝
Balise H1 non optimisée
Moyen
Le H1 actuel est "Gagne tes pâtes à tartiner" — orienté marketing/concours, il ne contient aucun mot-clé SEO lié au produit (pâte à tartiner protéinée, sans sucre, healthy, etc.).
✅
Données structurées présentes
OK
Les schémas Organization, WebSite et Product sont implémentés via JSON-LD. Le sitemap (4 fichiers) et le robots.txt sont également en place.
✅ Schema.org OK

Audit Performance

Problèmes techniques

⚡
10 CSS bloquant le rendu
Critique
10 fichiers CSS render-blocking empêchent l'affichage initial. Parmi eux : Google Fonts, base.css, et 8 fichiers de composants (cart, cart-drawer, progression...). Ils devraient être chargés en différé ou inlinés.
📦
Surcharge JavaScript
Critique
142 fichiers JS pour un total de 1.5 MB. Cela impacte fortement le Time to Interactive, surtout sur mobile. Beaucoup viennent des scripts tiers (Klaviyo, Facebook, Hotjar, GTM...).
🏗
DOM volumineux
Moyen
Le DOM contient 1 680 éléments avec une profondeur de 18 niveaux et un élément (body) ayant 122 enfants directs. Cela ralentit les calculs de style et les reflows.
📈
Layout Shift à la limite
Moyen
Le CLS est à 0.10 en données terrain (pile sur le seuil) et 0.13 en lab (au-dessus du seuil). Un shift de score 0.127 se produit à 411ms, probablement dû au chargement d'images ou de polices.
🔌
APIs dépréciées & cookies tiers
Moyen
Le pixel Facebook utilise AttributionReporting (API dépréciée). De plus, 3 cookies tiers sont détectés — ils seront bientôt bloqués par Chrome.
🔒
Score Best Practices bas (58/100)
Critique
Ce score faible est principalement causé par l'utilisation d'APIs dépréciées, les cookies tiers, et des pratiques non conformes aux standards modernes du web.

Plan d'action

Recommandations prioritaires

🔴 Priorité haute
1. Ajouter la meta description
Rédiger une meta description de 150-160 caractères incluant les mots-clés principaux : "pâte à tartiner protéinée", "sans sucre", "naturelle". Cela améliorera le CTR dans les résultats de recherche.
Impact : +15-25% CTR estimé
🔴 Priorité haute
2. Optimiser les attributs alt
Ajouter des textes alternatifs descriptifs aux 168 images concernées. Utiliser des descriptions incluant les noms de produits et mots-clés pertinents.
Impact : SEO images + Accessibilité
🔴 Priorité haute
3. Réduire les scripts tiers
Charger Klaviyo, Facebook, Hotjar et GTM en différé (defer/async). Évaluer si Chaty et Hotjar sont nécessaires simultanément. Économie potentielle : ~400-600 kB.
Impact : -30% temps de chargement
🟠 Priorité moyenne
4. Compléter les balises Open Graph
Ajouter une og:image attractive (1200x630px) et rédiger une og:description complète. Chaque partage sur les réseaux sociaux sera plus engageant visuellement.
Impact : Engagement social +
🟠 Priorité moyenne
5. Corriger les CSS render-blocking
Inliner le CSS critique (above-the-fold) et charger les 10 CSS de composants en différé avec media="print" onload. Google Fonts avec display=swap est déjà en place.
Impact : FCP amélioré
🟠 Priorité moyenne
6. Améliorer le maillage interne
Ajouter des liens internes vers les collections, le blog et les fiches produit. Réduire les liens externes non essentiels. Objectif : ratio 70% interne / 30% externe.
Impact : Distribution du PageRank
🟢 Priorité basse
7. Optimiser le H1 pour le SEO
Remplacer ou compléter le H1 avec des mots-clés ciblés tout en gardant l'aspect marketing. Exemple : inclure "pâte à tartiner protéinée" dans la structure de titres.
Impact : Pertinence sémantique
🟢 Priorité basse
8. Réduire le DOM et optimiser les images
Convertir les images restantes en WebP/AVIF via Shopify CDN. Lazy-loader les images below-the-fold. Simplifier la structure HTML du body (122 enfants directs).
Impact : CLS + Vitesse
